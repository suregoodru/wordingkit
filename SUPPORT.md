# Support

## Getting help

Use GitHub Issues for reproducible bugs, focused feature requests, and
documentation gaps.

Before opening an issue:

- update to the latest `main` commit or latest release;
- run `npm test` when the issue involves development setup;
- check whether the issue is provider-specific by trying a local Ollama mode;
- remove private text, API keys, tokens, customer data, and internal URLs from
  examples and screenshots.

## Not supported

This project does not provide support for:

- debugging private provider accounts or billing;
- recovering API keys or Raycast preferences;
- rewriting private documents shared through public issues;
- general Ollama, OpenAI, Anthropic, Groq, macOS, or Raycast support unrelated
  to WordingKit behavior.

## Security

Do not report vulnerabilities in public issues. Follow `SECURITY.md` for private
security reports.
