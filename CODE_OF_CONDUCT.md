# Code of Conduct

WordingKit is a small open-source project. Participation should stay focused,
respectful, and useful to maintainers and users.

## Expected behavior

- Be clear, specific, and considerate in issues, pull requests, and reviews.
- Assume public discussions may be read by people with different backgrounds,
  languages, and levels of experience.
- Keep disagreement technical and tied to the change being discussed.
- Respect privacy. Do not post private text, API keys, customer data,
  screenshots with secrets, or internal URLs.

## Unacceptable behavior

- Harassment, insults, threats, or personal attacks.
- Publishing private information without explicit permission.
- Repeated off-topic comments or spam.
- Attempts to bypass security reporting for sensitive issues.

## Enforcement

Maintainers may edit or remove comments, close issues, block users, or report
abuse to GitHub when participation harms the project or its contributors.

Security-sensitive reports should follow `SECURITY.md`, not public issue
threads.
